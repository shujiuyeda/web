1-1. Ghostty（ターミナルアプリ）
  • Ghosttyはコマンドを打ち込む画面（ターミナルアプリ）。Macのターミナルと違い、①画面を分割して複数同時表示できる ②テキスト表示が速い ③日本語が綺麗、という理由でこちらを使う
  • https://ghostty.org/ → Download → macOS → ユニバーサルバイナリー
  • ダウンロード後、アプリケーションフォルダにドラッグ
  • 日本語フォント設定コマンドを2つ実行（↓）
  • Ghosttyを起動して動作確認
  • 「"Ghostty.app"から"〇〇"フォルダ内のファイルにアクセスしようとしています」というダイアログが出たら → すべて「許可」をクリック（ダウンロード・書類・デスクトップなど使うフォルダの数だけ出る。初回のみ、全部許可でOK）
  • 以降のコマンドはすべてGhosttyで実行する（Macの標準ターミナルは使わない）
  • ターミナルはここでお役御免 → ターミナルのウィンドウに exit と入力してEnterを押すと完全に終了する（×ボタンで閉じるだけではプロセスが残ることがある）

Phase 2: プロジェクト設定

2-1. Obsidian（ファイル管理アプリ）
  • ObsidianはMarkdown（.md）ファイルを管理するアプリ。Claude Codeが書いてくれたログ・原稿・メモをここで確認・編集できる。iPhoneとも同期できる
  • Markdownとは：普通のテキストファイル（メモ帳で書けるようなもの）に「#」で見出し・「-」でリストなどの簡単な書式が使える形式。Claudeとのやりとりのログや設定ファイルはすべてこの.md形式で保存される
  • https://obsidian.md/ → 「Get Obsidian for macOS」をクリック → ダウンロードされた.dmgファイルを開く → アプリケーションフォルダにドラッグ
  • 「Create new vault」→ 名前は半角英数字で自由に決める（例: ccws・satoyumi-works・my-workspace など）。日本語はコマンドでトラブルの原因になるのでNG
  • 場所: 「Store in iCloud」を選ぶ → iCloud Drive内に自動保存され、iPhoneからも見られるようになる
  • 日本語化: Obsidianを起動したら、画面左下の ⚙️（Settings） をクリック → 「General」を開く → 「Language」のプルダウンで「日本語」を選択 → 「Relaunch」ボタンをクリック → Obsidianが再起動して全メニューが日本語になる
このフォルダでClaude Codeを起動（初回・フルパス）
cd ~/Library/Mobile\ Documents/iCloud\~md\~obsidian/Documents/（ボルト名） && claude

⚠️ （ボルト名） の部分は、Obsidianで作ったボルト名に書き換えてから実行すること（例: ccws → 自分でつけた名前）

ショートカット登録（一度だけ実行）— 次回から cc だけでOK
echo 'alias cc="cd ~/Library/Mobile\ Documents/iCloud\~md\~obsidian/Documents/（ボルト名） && claude"' >> ~/.zshrc && source ~/.zshrc

⚠️ こちらも （ボルト名） を自分のボルト名に書き換えてから実行する。登録後は cc とだけ打てばClaude Codeが起動する

⚠️ Claude Codeは起動したフォルダがワークスペースになる。必ずこのフォルダに移動してから起動すること。セキュリティチェックが出たら「1. Yes, I trust this folder」を選ぶ
2-2. Claude の環境を初期化する
  • 起動できたら、まず /init と入力してEnter → Claudeがプロジェクト構造を分析して CLAUDE.md の雛形を自動生成してくれる
  • CLAUDE.mdとは: Claudeへの「指示書」。言語（日本語で話して）・役割（ライターとして）・よく使うワークフロー・ルールなどを書いておくと、次回起動時から自動で読み込まれる
  • 最初は /init で生成した雛形で十分。使いながら少しずつ育てていけばOK
  • /init で生成された雛形をObsidianで開いて編集する（Obsidianから普通のテキストとして編集できる）
  • 最低限書くと効くこと:
    • 言語: 「日本語で返答して」
    • 役割: 「私はライター。取材・執筆・編集が主な業務」
    • よく使う操作: 「ログは logs/YYYY-MM-DD.md に書く」「原稿は drafts/ に保存する」など
  • 書かなくていいこと: Markdownの書き方・Gitの基本コマンドなど、Claudeが元々知っていること。「知ってて当然」な内容は省く
  • 完璧を目指さなくてOK。使いながら「こう動いてほしかった」と気づいたらその都度1行追加するだけで育っていく

Phase 5: 「今日もコレカラ」バナー自動化

5-1. .aiファイルの場所を確認
  • バナー元ファイル（.ai）の保存場所をさとゆみさんに聞く
  • Vol番号を管理しているファイル（スプレッドシート等）も確認
5-2〜5-4. スクリプト作成 → テスト → スキル登録
  • 初回のみ現在のVol番号をセット（↓）
  • スクリプト本体を scripts/update_banner.sh に保存（↓）
  • 動作: Illustratorで.aiを開く → Vol番号・日付を書き換え → PNGで書き出し → 閉じる
  • CLAUDE.mdに /koreka スキルを登録 → 「今日もコレカラ作って」で呼び出し可能に
初回: 現在のVol番号をセット（例: 683）
echo "683" > sns/vol_number.txt
スクリプト本体（scripts/update_banner.sh に保存）
#!/bin/bash
VOLFILE="sns/vol_number.txt"
AIFILE="$HOME/Desktop/今日もコレカラバナー.ai"  # ← 実際のパスに変更

VOL=$(cat "$VOLFILE")
NEXT_VOL=$((VOL + 1))
echo "$NEXT_VOL" > "$VOLFILE"

DATE=$(date '+%Y/%-m/%-d')
OUTFILE="sns/今日もコレカラ_$(date '+%Y%m%d').png"

osascript <<EOF
tell application "Adobe Illustrator"
    open POSIX file "$AIFILE"
    set doc to current document
    repeat with tf in every text frame of doc
        if text of tf contains "Vol." then
            set text of tf to "Vol.$VOL　$DATE"
        end if
    end repeat
    export doc to POSIX file "$(pwd)/$OUTFILE" as PNG
    close doc saving no
end tell
EOF

echo "書き出し完了: $OUTFILE（Vol.$VOL / $DATE）"
実行テスト（毎日これだけ）
bash scripts/update_banner.sh
